# 内政与粮草系统需求（势力共享府库版）

> 文档版本：V2.0
> 更新日期：2026-07-16
> 状态：当前正式规则
> 适用工程：`godot/` 下的 Godot 4.7 .NET / C# 版本

## 1. 已确定的设计边界

1. 金钱和粮草均为势力级通用资源，归入所属势力的唯一府库。
2. 不建立城市独立金库、城市独立粮库、中央与城市调拨、道路运输损耗或自动上缴系统。
3. 城池保留独立的人口、发展、驻军、设施、太守、治理方针、城市定位、建设队列、城务额度和台账。
4. 城务额度逐城计算与消耗；一座城用完城务不会占用另一座城的额度。
5. 城市产出直接进入势力府库，城市维护、城务、建设、出征和军团回粮也直接与势力府库结算。
6. `CityData.Gold/Food` 只作为剧本开局库存来源；进入新游戏后不再作为可流转的城市余额。
7. 玩家可以逐城选择亲自治理或方针委任，但不存在全势力托管或全 AI 演进。

本规则为当前项目的确定方向。后续功能不得重新引入本地粮库、资源调拨或为旧存档保留相关兼容层。

## 2. 资源所有权与流向

### 2.1 开局府库

新游戏按玩家选择的势力汇总全部初始城池库存：

```text
势力初始金 = 所辖城市 CityData.Gold 之和
势力初始粮 = 所辖城市 CityData.Food 之和
```

难度只在汇总后调整玩家开局资源：

| 难度 | 金钱与粮草倍率 |
| --- | ---: |
| 休闲 | 1.20 |
| 标准 | 1.00 |
| 困难 | 0.85 |

### 2.2 每月资源闭环

```text
各城人口、发展与设施计算月产出
  → 直接增加所属势力府库
  → 从同一势力府库支付设施与驻军维护
  → 城务、建设和出征在玩家确认时即时扣除
  → 军团撤回或战斗结束时，剩余军粮回到同一势力府库
```

同一笔城市产出只能进入势力府库一次。城市台账记录该城造成的收入和支出，但台账不是第二套资源账户。

### 2.3 不存在的操作

- 不提供“中央调往城市”或“城市上缴中央”。
- 不按道路距离收取金粮运输损耗。
- 不要求偏远城先收到粮食才能执行城务。
- 不把城市初始 `Gold/Food` 持续显示为仍可消费的本地库存。

## 3. 城市月度产出与维护

### 3.1 金钱产出

```text
基础金钱 = 人口 × 商业 ÷ 12,000
市集产出 = 市集等级合计 × 180
月产金 =（基础金钱 + 市集产出）× 治安修正 × 商埠定位 × 朝堂收入修正
```

治安低于 40 时修正为 0.72；治安 40–59 时为 0.88；其他情况为 1.00。完成转型的商埠提供 1.15 倍金钱产出。

### 3.2 粮食产出

```text
基础粮食 = 人口 × 农业 ÷ 27,000
设施粮食 =（灌溉渠等级合计 + 粮仓等级合计）× 120
月产粮 =（基础粮食 + 设施粮食）× 季节修正 × 粮仓定位 × 朝堂收入修正
```

季节修正：

| 月份 | 修正 |
| --- | ---: |
| 1–2 月 | 0.88 |
| 8–10 月 | 1.25 |
| 其他月份 | 1.00 |

完成三个月转型的“粮仓”城市提供 1.15 倍粮食产出。朝堂粮食收入加成总上限为 20%。

平衡目标：自然农业承担驻军维护的主要部分，粮食设施用于补足缺口并支撑扩军，不允许单座一级设施远高于整座普通城市的自然农业产出。以标准刘备开局为基线，自然月产粮约 348、驻军月耗约 592；两座一级粮食设施合计增加 240 后接近平衡。

### 3.3 月度维护

```text
设施金钱维护 = 全部设施等级合计 × 20
驻军粮食维护 = Max(80, Round(驻军 ÷ 14 × 朝堂后勤修正))
```

朝堂后勤减免总上限为 15%，同时作用于驻军与在外军团的军粮消耗。

### 3.4 粮荒

某城结算时，势力府库无法支付该城完整驻军粮耗，则该城当月：

- 民心降低 5。
- 治安降低 3。
- 驻军减少 `Max(100, 驻军 ÷ 25)`，驻军最低保留 1,000。
- 战略年表记录粮荒。

势力余粮低于全势力一个月驻军维护时，所属城市显示“缺粮”；低于两个月维护时可触发低粮事件。

## 4. 城务额度与治理

### 4.1 每城独立额度

```text
基础额度 = 2
称职太守 = +1（政治达到 75 且能够履职）
郡府 = +1
动荡或缺粮 = -1
每城额度 = Clamp(基础额度 + 加成 - 惩罚, 1, 4)
```

- 每座城分别计算、分别消耗、每月分别重置。
- 普通城务和设施开工消耗本城 1 点城务。
- 同一武将通常每月只能执行一次普通城务。
- 城务使用本城武将和本城状态计算效果，但费用从势力府库扣除。

### 4.2 亲自治理与方针委任

- 亲自治理：玩家选择命令和执行武将后立即结算；月末不会自动使用剩余额度。
- 方针委任：月末由太守根据方针、城市状态、可用武将和势力府库选择普通城务。
- 委任只使用当前城市的剩余城务额度。
- 委任不能处理人才任命、外交、出征、军团命令或事件选择。

可选方针为均衡治理、休养生息、富民通商、屯田积粮、整军备战和巩固新土。

## 5. 城务命令与粮食费用

| 命令 | 势力府库粮耗 | 主要效果 |
| --- | ---: | --- |
| 劝课农桑 | 0 | 提高本城农业，长期增加月产粮 |
| 振兴商业 | 0 | 提高本城商业 |
| 整顿治安 | 0 | 提高本城治安 |
| 修缮城防 | 0 | 提高城防、城墙和城门耐久 |
| 征募士卒 | 600 | 人口转为驻军，降低民心与平均训练 |
| 操练兵马 | `Max(400, 驻军 ÷ 25)` | 提高本城驻军训练 |
| 寻访人才 | 0 | 发现人才；无人时提高文化 |
| 赈济百姓 | 1,200 | 提高民心并降低疲敝 |

所有命令另有金钱费用；执行前必须同时检查本城城务、武将状态、势力金粮和人口条件。

## 6. 粮食设施

### 6.1 灌溉渠

- 建设费用：金 800、粮 300。
- 工期：2 个月。
- 每级增加 120 月产粮。
- 每级强化劝课农桑效果。

### 6.2 粮仓

- 建设费用：金 1,000、粮 400。
- 工期：2 个月。
- 每级增加 120 月产粮。
- 每级强化劝课农桑与赈济效果。

同一城市不能重复建设同类设施。设施最多三级，升级提高同一核心作用，不新增本地库存、容量或调拨规则。

## 7. 出征与军团军粮

### 7.1 装粮

- 出征军粮由玩家在出征界面指定。
- 军粮直接从势力府库扣除并转入新军团的 `ArmyData.Food`。
- 出发城只负责提供驻军和武将，不拥有独立可扣除的本地粮库。

### 7.2 行军消耗

```text
每次行军军粮消耗 = Max(100, Round(军团兵力 ÷ 25 × 朝堂后勤修正))
```

- 有粮时每次推进 30 天，士气降低 1。
- 粮草耗尽时每次只推进 18 天，士气降低 12。
- 无粮进入战斗时，军团状态战力乘 0.72。

### 7.3 军粮回库

以下情况必须把军团全部剩余军粮一次性归还所属势力府库，并把军团携粮清零：

- 玩家主动撤兵。
- 停战召回。
- 野战结束后回营。
- 攻城胜利并驻入新城。
- 攻城失败并返回出发城。

围城僵持不属于军团结束状态，剩余粮草继续保留在军团并在后续月份消耗。清零是防止同一批剩余军粮被重复回收的必要条件。

## 8. 战利品与事件

- 攻陷城市后，从败方势力府库缴获其当前金钱的 10%，最多 1,200金。
- 攻陷城市后，从败方势力府库缴获其当前粮食的 10%，最多 4,000粮。
- 战利品直接进入胜方势力府库，与攻城军团剩余军粮回库分别计算。
- 城市事件产生的金粮变化直接作用于该城所属势力府库，同时写入该城台账。

## 9. AI 规则

玩家逐城委任和非玩家势力 AI 使用与玩家相同的城务、费用、建设时间和势力府库：

- 粮食覆盖不足时提高农业、粮仓和灌溉渠优先级。
- AI 出征后必须至少保留四个月驻军粮耗或 4,000粮，取较高者。
- AI 出征真实扣除势力粮草，不能免费生成军粮。
- AI 不得跨城借用城务额度，也不能让同一武将在多城同时行动。

## 10. 界面与台账

### 10.1 势力内政总览

- 顶部显示势力府库当前金粮。
- 每座城市显示独立城务、状态、太守、定位、方针、驻军和本城预计月度贡献。
- 本城月度贡献等于该城产出减去该城维护，但实际资金始终进入同一势力府库。

### 10.2 单城页面

- 显示势力府库，不显示可消费的城市金粮余额。
- 显示本城预计产出、维护和净贡献。
- 城务确认区明确写明“势力府库金/粮”费用。
- 最近台账按城市记录经济、城务、建设和事件来源。

### 10.3 天下地图

右上资源栏显示当前势力金粮及结束本月后的预计净变化。预估必须与正式月末结算使用同一套运行逻辑。

## 11. 验收标准

- [ ] 金钱和粮草只有势力府库一个运行时账户，城市初始库存只用于开局汇总。
- [ ] 不存在城市与中央之间的调拨入口。
- [ ] 每座城市拥有独立城务额度，一城行动不消耗另一城额度。
- [ ] 城市产出和维护直接结算势力府库，并写入对应城市台账。
- [ ] 标准刘备开局自然粮产承担至少一半驻军维护，但尚未自然盈余。
- [ ] 每座初始城市各增加一座一级粮食设施后，总产粮与驻军维护相差不超过 10%。
- [ ] 一级灌溉渠或粮仓增加 120 月产粮，不再远高于城市自然农业。
- [ ] 劝农、粮食设施、粮仓定位、季节和朝堂加成共同进入同一产粮公式。
- [ ] 主动撤兵、停战召回、野战回营以及攻城胜负均只回收一次剩余军粮。
- [ ] 回库后军团 `Food` 为 0，不会在失效军团对象中残留可重复计算的粮食。
- [ ] AI 城务、建设和出征与玩家遵守相同的势力府库约束。
- [ ] 资源预估与真实结束回合结果一致。
- [ ] `./tools/run-godot.sh check` 与 `./tools/run-godot.sh test` 通过。

## 12. 完成定义

玩家应当清楚感受到：自己在发展不同城市，但所有城市共同支撑同一个势力府库；扩军和出征消耗全势力粮草，农业、设施、城市定位和朝堂任命共同决定长期承载力；任何结束行动的军团都不会吞掉或重复归还剩余粮草。
