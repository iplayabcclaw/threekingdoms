# 内政系统深化需求（城池独立管理版）

> 文档版本：V1.0  
> 编写日期：2026-07-14  
> 状态：核心闭环已实现；区域托管与道路运输深化保留为后续切片  
> 适用工程：`godot/` 下的 Godot 4.7 .NET / C# 正式版本

## 1. 文档目的

本需求用于重构当前内政玩法，使经营对象从“势力共享的少量操作”变为“每座城都有独立经营状态与本地执行能力”，同时控制中后期操作负担。

本次改造需要达成以下结果：

1. 每座城单独保存金、粮、驻军、武将、设施、发展状态和当月城务额度。
2. 城池的普通内政不再消耗势力共享的三次操作；一座城用完额度不影响其他城继续治理。
3. 城池收入先进入本城府库，本地命令优先消耗本城资源，避免“远方孤城瞬间使用全国钱粮”。
4. 玩家可以逐城精细管理，也可以给稳定城池设置方针，由太守自动完成日常事务。
5. 丰富度主要来自城市差异、命令取舍、武将安排、设施组合和事件反馈，不新增大量资源种类与重复按钮。
6. 内政结果能够继续影响征兵、守城、出征补给、人才、灾害和占领后的恢复，而不是独立的数值小游戏。

本文是内政系统的专项详细需求。若与 `docs/01-requirements.md` 第 8、26、27、33.1 节中的较早概述冲突，以本文为准。

## 2. 当前实现审计

### 2.1 已有基础

当前 `CityData` 已经包含以下逐城字段：

- 人口、农业、商业、治安、民心、城防、文化、训练与疲敝。
- 城池金、城池粮、驻军。
- 太守、驻城武将、设施槽、已建设施和单条建设队列。
- `ActionSlots` 城池行动槽。

当前城池页面已经提供：

- 发展农业、振兴商业、整顿治安、修缮城防。
- 征募士卒、操练兵马、寻访人才、赈济百姓。
- 十类设施的建设、升级和修缮。
- 势力府库与城市府库之间的手动调拨。
- 驻城武将选择和官职调整。

因此，本次不需要推翻现有城池模型，重点是统一资源流、行动规则、自动治理和界面层级。

### 2.2 当前真实规则

| 项目 | 当前实现 | 造成的问题 |
| --- | --- | --- |
| 初始行动槽 | 剧本中每城通常为 2 | 已经按城保存，但界面缺少全城视角，玩家难以感知“各城独立” |
| 月初行动槽 | `1 + 人口 / 100000 + 文化 / 50`，范围 1–6 | 人口与文化同时增长时额度偏多；也无法体现太守、治安和设施 |
| 普通内政费用 | 主要从势力总府库扣除 | 本城没钱也能建设或赈济，城市府库缺乏实际意义 |
| 建设与修缮费用 | 从势力总府库扣除 | 城池资源、运输距离和前线压力没有形成约束 |
| 征兵 | 扣势力金和军备，直接增加驻军 | 不扣本城人口与本城粮，征兵缺少地方代价 |
| 训练 | 只扣势力金 | 不消耗本地粮食，驻军规模也不影响训练成本 |
| 月度产出 | 每城产出的金粮直接进入势力总府库 | 城市金粮不参与正常经济循环，逐城经营流于表面 |
| 军队维护 | 从势力总粮统一结算 | 被阻断或偏远城池仍可无条件享受全国粮食 |
| AI 内政 | 敌城每月固定加农业、商业和驻军 | 不使用资源、武将、行动槽与建设规则，玩家和 AI 不同规则 |
| 玩家托管 | 只选择全势力最低发展的一座城执行一次 | 多城势力的大量城池不会获得正常治理，且不符合逐城管理 |
| 命令展示 | 八个命令在同一行平铺 | 信息拥挤，玩家只看到按钮，难以判断本城当前最重要的问题 |

### 2.3 核心问题

当前系统已经“有每城数据”，但还没有形成真正的每城经营闭环：

```text
城市产出
  → 直接进入势力府库
  → 玩家使用势力府库在任意城市执行命令
  → 城市自己的金粮大部分时间不参与规则
```

目标闭环应调整为：

```text
城市人口、发展与设施产生本地收入
  → 进入本城府库
  → 支付本城维护和本城命令
  → 形成盈余或缺口
  → 通过道路向势力中枢上缴、申请调拨或请求邻城支援
```

## 3. 设计目标与边界

### 3.1 核心原则

1. **城池是独立经营单位**：每座城有自己的额度、资源、人员、计划和月报。
2. **势力负责战略，城池负责日常**：势力级资源与政令用于跨城调度、外交和战争，不替代普通城市经营。
3. **增加取舍，不增加流水账**：同一城每月只需决定少量重点，稳定城市可以自动治理。
4. **城市有差异但规则统一**：通过地理、人口、特产、设施和前线位置形成差异，不为 33 座城分别发明一套规则。
5. **每个数值都连接其他系统**：农业影响粮食，商业影响金钱，治安影响损耗，民心影响征兵，训练和城防进入战斗。
6. **结果必须可解释**：命令执行前展示费用、预计收益和副作用，月报展示实际变化及原因。

### 3.2 明确不做

- 不新增木材、石料、铁矿、布匹等大量基础物资；首版继续以金、粮、军备为主要资源。
- 不做居民逐户模拟、官员编制表、税率明细表或生产链排班。
- 不要求玩家每月依次进入所有城池并点完额度。
- 不允许一个“全势力发展农业”按钮无差别作用于全部城市。
- 不让设施产生几十种小百分比加成；设施优先解锁能力、改变上限或形成明确专长。
- 不把行动额度作为可囤积资源；当月未使用额度月末清零。

## 4. 管理层级

### 4.1 城务额度：每座城独立

原“行动槽”统一改名为“城务额度”，强调它属于当前城池，而不是整个势力。

首版规则：

```text
基础额度 = 2
太守称职 = +1（太守政治达到 75，且健康、状态正常）
郡府加成 = +1（郡府达到指定等级）
混乱惩罚 = -1（治安低于 30 或城市处于严重灾害）

每城每月城务额度 = Clamp(基础额度 + 加成 - 惩罚, 1, 4)
```

- 每座城分别计算、分别消耗、分别重置。
- 徐州用完 3 点城务额度后，下邳仍可使用自己的 3 点。
- 城务额度只用于本地命令，不用于外交、跨城调兵或建立战役。
- 普通命令消耗 1 点；大型建设在开工时消耗 1 点，后续月份自动施工，不重复占用额度。
- 太守空缺时仍保留 1 点最低额度，但只能执行赈济、巡查、修缮等应急命令。
- 不再按人口线性增加到 5–6 点，避免大城市每月出现过多点击。

### 4.2 势力政令：只管理跨城战略

势力政令与城务额度必须分开显示、分开结算。

势力政令用于：

- 城池之间调拨大额金粮。
- 跨城调动武将与驻军。
- 更换太守或改变区域委任。
- 主动外交、情报专项和战役建立。
- 修改城市长期定位或下达紧急征调。

势力政令不用于：

- 单城发展农业或商业。
- 单城巡查、赈济、训练和普通征兵。
- 已批准建设的每月施工。
- 太守依据既定方针执行的常规内政。

这样可以保留“大势力不能无限亲自干预”的行政约束，又不会出现“整个势力只有三次内政操作”的问题。

### 4.3 手动治理与委任治理

每座城拥有独立治理方式：

| 方式 | 玩家操作 | 执行规则 | 适用场景 |
| --- | --- | --- | --- |
| 亲自治理 | 玩家为本月逐项下令 | 立即占用本城额度，并由指定武将执行 | 都城、前线、新占城、危机城 |
| 方针委任 | 玩家选择城市方针和资源底线 | 月末由太守使用剩余额度自动执行 | 稳定后方城市 |
| 区域托管 | 玩家为若干城设共同目标与边界 | 各城仍使用自己的额度和府库，区域长官只负责排序与调拨建议 | 中后期多城势力 |

委任不是全势力凭空获得收益。自动命令仍必须满足：

- 当前城市有剩余城务额度。
- 本城府库能支付费用。
- 城内存在符合条件的太守或执行武将。
- 没有违反玩家设定的金粮底线、禁止征兵或禁止拆借规则。

## 5. 城池经营模型

### 5.1 保留的核心属性

| 属性 | 主要作用 | 主要变化来源 |
| --- | --- | --- |
| 人口 | 税收、粮食需求、征兵上限与城市规模 | 民心、治安、灾害、征兵、战争 |
| 农业 | 本地粮食产出 | 劝农、灌溉、灾害、季节 |
| 商业 | 本地金钱产出 | 通商、市集、治安、战争 |
| 治安 | 腐败、盗匪、运输损耗与事件风险 | 巡查、驻军、民心、灾害 |
| 民心 | 人口增长、征兵代价与占领整合 | 赈济、税负、征兵、事件 |
| 城防 | 守城阶段和设施受损概率 | 修城、城墙、战斗破坏 |
| 文化 | 人才发现、治理效率与学府能力 | 兴学、学宫、人才事件 |
| 训练 | 本城驻军战斗质量 | 训练、战损、新募兵补入 |
| 疲敝 | 产出与命令效率惩罚 | 连续征兵、灾害、围城、休养 |

首版不新增更多常驻 0–100 条目。城市差异优先通过“状态”和“定位”表达。

### 5.2 城市状态

每座城同一时间最多拥有一个主要状态：

| 状态 | 触发示例 | 可见影响 |
| --- | --- | --- |
| 安定 | 治安、民心均不低于 60 | 正常产出，可安全委任 |
| 繁荣 | 连续 3 月安定且金粮有盈余 | 商业或人口获得小幅成长，事件偏正面 |
| 缺粮 | 月末粮食不足以支付需求 | 民心下降、训练和征兵受限 |
| 动荡 | 治安低于 40 或刚发生严重事件 | 税收损耗、运输风险上升 |
| 疲敝 | 疲敝达到 60 | 全部发展命令效率降低 |
| 前线 | 相邻道路连接敌城或敌军逼近 | 守备权重上升，商业受压，允许紧急动员 |
| 整合中 | 城池刚易主 | 产出、征兵和上缴受限，必须选择占领政策 |

状态由既有数值推导，不单独增加需要玩家维护的新资源条。

### 5.3 城市定位

稳定城市可以选择一个长期定位：

| 定位 | 核心能力 | 明确短板 |
| --- | --- | --- |
| 粮仓 | 可设置战略储粮，荒年和补给能力更强 | 商业成长较慢 |
| 商埠 | 可建立通商路线，金钱与调拨效率更高 | 驻军维护和征兵效率一般 |
| 军镇 | 征兵、训练、整补与守城能力更强 | 民心恢复慢，粮耗较高 |
| 学府 | 人才发现、文化和太守治理更强 | 直接金粮产出较弱 |
| 枢纽 | 降低跨城调拨损耗并支援相邻城市 | 自身专项产出不突出 |

- 定位由设施与连续执行的命令逐步形成，不在开局永久锁死。
- 每城只能有一个主要定位。
- 更换定位需 3 个月过渡期和一次势力政令，防止根据当月收益频繁切换。
- 定位首版只提供 1 个主动能力和 1 个明确被动规则，不堆叠多条小加成。

## 6. 本地资源与经济闭环

### 6.1 账户划分

保留两级账户，但职责重新定义：

| 账户 | 保存内容 | 主要收入 | 主要支出 |
| --- | --- | --- | --- |
| 城市府库 | 城池金、城池粮 | 本地税收、农业产出、贸易收入、调拨 | 本地内政、设施、驻军维护、赈济、征兵训练 |
| 势力府库 | 中央金、中央粮、威望、军备 | 城市上缴、战利品、外交收入 | 外交、跨城运输、战略工程、直属军团和紧急拨款 |

### 6.2 月度结算顺序

每座城按以下固定顺序独立结算：

```text
1. 计算本地金粮产出
2. 结算设施与城市状态修正
3. 支付驻军、设施和官员维护
4. 执行已排定的自动城务
5. 补足城市最低储备
6. 将允许上缴的盈余转入势力府库
7. 生成该城月报与资源台账
```

- 同一笔收入不能同时完整增加城市府库和势力府库。
- 所有移动必须形成一进一出的台账记录。
- 城市缺粮只影响该城及依赖它的军团，不直接让全国所有城市同时进入粮荒。
- 势力府库有粮不代表偏远城市自动获得粮食；必须存在调拨命令和可用道路。

### 6.3 城市储备与自动上缴

每城可设置简单的金粮策略：

- 最低金储备。
- 最低粮储备。
- 盈余上缴比例：不上缴、25%、50%、75%。
- 是否允许紧急支援相邻己方城市。

初始推荐值由城市定位自动给出，玩家通常不需要逐项调整。前线和新占城市默认不上缴粮食，后方商埠默认上缴较多金钱。

### 6.4 调拨规则

- 同城内政只消耗本城资源，不消耗势力政令。
- 城市与势力府库之间的大额调拨消耗 1 点势力政令。
- 调拨需要经过道路并产生基础运输损耗；首个切片可以按相邻城、非相邻城两档计算，不做逐车队模拟。
- 道路被敌军截断时，调拨不能完成，并在地图与月报中标记原因。
- 为避免玩家频繁搬运，允许设置一次性调拨和“低于底线自动申请”两种方式。

## 7. 内政命令

### 7.1 命令分组

保留当前八类命令，但界面按四组呈现，默认只展开与当前问题最相关的一组。

| 分组 | 命令 | 主要收益 | 主要代价或风险 |
| --- | --- | --- | --- |
| 生产 | 劝课农桑、振兴商业 | 农业或商业成长 | 消耗金；连续开发增加少量疲敝 |
| 民生 | 整顿治安、赈济百姓 | 治安、民心和灾后恢复 | 巡查需要合适武将；赈济消耗本城粮 |
| 军备 | 修缮城防、征募士卒、操练兵马 | 城防、驻军、训练 | 消耗人口、金粮、军备或民心 |
| 人才 | 寻访人才 | 发现武将、宝物线索或地方情报 | 结果不保证立刻成功，依赖智力与文化 |

设施建设作为独立“工程”页签，不与普通命令挤在同一排。

### 7.2 通用执行规则

每个命令必须包含：

- 执行城池。
- 执行武将。
- 消耗的本城城务额度。
- 本城金粮、人口或军备费用。
- 预计收益范围。
- 至少一个会影响结果的武将能力。
- 可能产生的副作用。

通用效率建议：

```text
基础效果
× 武将能力系数
× 太守协同系数
× 设施与定位系数
× 城市状态系数
× 小幅确定性波动
```

- 小幅波动必须使用存档内可复现的随机状态。
- 执行前展示区间，例如“农业预计 +4～6”。
- 执行后展示实际值及前三项主要修正，例如“陈登政治优秀 +18%，疲敝 -10%”。

### 7.3 命令取舍

为避免所有命令都只是“花钱加数值”，首版加入以下直接联动：

- 劝农可提高农业，但在农忙月份效果更好；连续三月劝农会增加疲敝。
- 经商需要足够治安；治安过低时收入会被盗匪和腐败截留。
- 巡查可提高治安，但武力过高、魅力过低的执行者可能只提高治安而降低少量民心。
- 赈济从本城粮仓扣粮；缺粮时不能用中央库存绕过道路直接赈济。
- 征兵同时消耗人口、金、粮和军备，并降低民心；征兵越接近本城人口上限，代价越高。
- 训练费用随驻军人数增加，消耗本城粮；新募兵会拉低平均训练度。
- 修城在城防较高时收益递减；刚经历攻城的城市修复效率更高。
- 寻访不会每次都获得武将，也可能获得宝物线索、地方情报或短事件，避免空结果。

## 8. 设施与建设

### 8.1 建设规则

- 每座城保持独立设施槽和独立建设队列。
- 开工消耗 1 点本城城务额度，并从本城府库扣除首期费用。
- 工程后续按月自动推进，不重复消耗城务额度。
- 施工武将必须留在本城；调离后工程暂停或由太守以较低效率接管。
- 城池受围、缺粮或治安过低时施工效率下降，并明确显示预计延期。
- 修缮和升级同样从本城府库扣费，不允许无视城市财力直接使用全国资金。

### 8.2 设施角色

现有十类设施继续复用，并把功能集中到清晰角色：

| 设施 | 首要作用 |
| --- | --- |
| 灌溉渠 | 提高农业命令稳定性，减轻旱灾影响 |
| 粮仓 | 提高安全储粮上限，降低腐坏与赈济损耗 |
| 市集 | 解锁商埠能力与相邻通商 |
| 工坊 | 生产或修复军备，支撑征兵与攻城器械 |
| 兵营 | 提高征兵上限并降低新兵整编时间 |
| 校场 | 提高训练效率，减少大驻军训练费用 |
| 城墙 | 提高攻城战外墙阶段防御与耐久 |
| 学宫 | 提高文化、人才发现和太守培养能力 |
| 医馆 | 降低灾害人口损失并加快武将恢复 |
| 郡府 | 提高城务额度或委任治理效率 |

每类设施首版最多 3 级。升级应强化同一个核心作用，不在每级新增一套子系统。

## 9. 太守、武将与治理

### 9.1 太守作用

太守不是名字展示，必须参与以下规则：

- 决定是否获得额外城务额度。
- 为本城全部命令提供政治与魅力相关的基础协同。
- 在方针委任模式下选择执行武将和命令顺序。
- 处理低风险城市事件；重大事件仍交给玩家。
- 影响治安恢复、腐败损耗和新占城市整合。

### 9.2 执行武将

- 每条手动命令必须指定一名当前驻城且可用的武将。
- 同一武将每月最多执行 1 条普通城务；太守可额外处理 1 条委任城务，但会增加疲劳。
- 出征、被俘、重伤或调动中的武将不能执行城务。
- 五维各有明确适用场景，避免所有内政都只看政治：
  - 政治：农业、建设、教化。
  - 魅力：商业、赈济、征兵民心代价。
  - 武力：巡查、治安威慑。
  - 统率：城防、训练、征兵整编。
  - 智力：寻访、工程规划、事件识别。
- 武将特性只改变少数具体规则，例如“爱民”降低征兵民心损失，“屯田”提高劝农效果。

### 9.3 太守空缺与不称职

- 太守空缺时城市最低仍可运行，但只有 1 点城务额度，不能开启方针委任。
- 太守政治较低不会让城市完全停摆，但自动治理更保守，且无法获得额外额度。
- 玩家更换太守消耗势力政令；太守跨城赴任需要道路时间，不允许瞬间到任。
- 前任离开至新任到达期间由城内政治最高的文官临时署理，但不能改变长期定位。

## 10. 自动治理与减负

### 10.1 城市方针

每城可选择一个方针：

| 方针 | 自动优先级 | 强制边界 |
| --- | --- | --- |
| 均衡治理 | 先处理红色异常，再补最低属性 | 不主动征兵，不动用储备粮 |
| 休养生息 | 民心、治安、农业、降低疲敝 | 禁止征兵和大型建设 |
| 富民通商 | 商业、治安、市集相关建设 | 保留最低粮储备 |
| 屯田积粮 | 农业、粮仓、储粮 | 保留最低金储备 |
| 整军备战 | 训练、征兵、城防、军备设施 | 不让民心低于玩家设置值 |
| 巩固新土 | 民心、治安、修复、整合 | 禁止上缴粮食 |

自动治理每月最多使用该城剩余城务额度，不会从其他城市借额度。

### 10.2 异常优先

即使城市设置了经济方针，以下情况也应暂停普通计划并请求处理：

- 粮食无法支付下月基本维护。
- 敌军预计两个月内抵达且守军不足。
- 治安或民心低于 30。
- 灾害、叛乱、围城或重大设施损坏。
- 太守离任、死亡、被俘或无法履职。
- 自动计划需要突破玩家设定的金粮底线。

玩家可在城池总览中一键接受太守建议，也可进入城市改为手动治理。

### 10.3 月报

月末不连续弹出 10 张城市窗口，而是生成一张“内政月报”：

- 顶部只显示 3–5 条最重要异常。
- 中部展示各城金粮净变化、执行命令和建设进度。
- 正常且无变化的稳定城市折叠为一行。
- 点击任一城市可展开完整台账与原因。
- 玩家可以按“异常、前线、缺粮、建设中、新占”筛选。

## 11. 城池管理界面

### 11.1 势力内政总览

主导航“内政”首先进入全城总览，不直接跳入某一座默认城市。

每座己方城市以一行或卡片展示：

- 城名、太守、城市状态和定位。
- 本月城务额度，例如“2/3”。
- 金、粮及预计下月净变化。
- 人口、驻军和危险标记。
- 当前方针、在建工程和异常提醒。
- “进入治理”按钮。

支持按地区、状态、前后方、金粮缺口和是否已下令排序。

### 11.2 单城页面

单城页面分为四个稳定区域：

1. **城池概况**：状态、定位、太守、本月额度与关键警告。
2. **收支与储备**：本城金粮、预计产出、维护、储备线和上缴策略。
3. **本月城务**：四组命令、执行武将、费用、预计效果和已排计划。
4. **设施与驻城人员**：设施队列、驻城武将、任命和治理建议。

当前八个命令按钮不再横向挤在同一行。玩家先选择分组，再查看 2–3 个相关命令。

### 11.3 地图入口

- 从天下地图点击己方城池可直接打开该城。
- 城池摘要显示“本月剩余城务额度”和首要问题。
- 单城页面提供上一城、下一城按钮，顺序与总览筛选一致。
- 非己方城池只显示情报，不出现内政操作。

### 11.4 信息反馈

每条命令的确认区至少展示：

```text
执行：陈登
消耗：本城城务 1，本城金 250
预计：农业 +4～6
影响：政治优秀；灌溉渠加成；城市疲敝使效果降低
执行后：本城剩余城务 1/3，本城金 3,720
```

不得只提示“府库不足”，必须写明是“徐州城府库金不足”还是“势力中央府库不足”。

## 12. 与其他系统的连接

### 12.1 战争与补给

- 出征军粮优先从出发城粮库装载。
- 驻军维护由驻扎城市支付。
- 城市缺粮会降低训练和守军士气，并可能产生逃兵。
- 前线城、粮仓与枢纽可以成为军团补给源。
- 攻城破坏会降低城防、人口、设施状况和本地府库，不只改变归属。

### 12.2 人才

- 人才只能在实际所在城执行当地命令。
- 学府和高文化城市更容易发现、培养人才。
- 太守与执行武将的功勋记录具体来源，例如“治理徐州三月”“平息下邳动荡”。
- 长期把不擅治理的武将安排为太守应体现为效率问题，而不是隐藏惩罚。

### 12.3 占领与整合

- 新占城保留战后剩余的本地金粮和受损设施。
- 易主后必须选择安抚、军管或自治政策。
- 整合期间仍有自己的城务额度，但部分命令锁定或效率下降。
- 新占城默认不上缴资源，避免刚占领即成为无代价提款机。

### 12.4 事件与灾害

- 城市事件读取本城状态与资源，不默认作用于所有城市。
- 旱灾影响农业与粮食，盗匪影响治安与运输，瘟疫影响人口与武将健康。
- 同一事件至少提供“花本地资源解决”“请求中央支援”“承受后果”三类思路中的两类。
- 委任城市只自动处理低风险事件；会改变民心、人口或战略储备的选项必须由玩家确认。

## 13. AI 规则

玩家与 AI 必须使用同一套城池资源和城务约束。

每个 AI 城市每月执行：

```text
读取城市状态与势力战略
→ 生成候选命令
→ 排除资源不足、武将不可用和违反储备线的命令
→ 评估缺粮、敌军、动荡、发展与建设需求
→ 按该城城务额度选择命令
→ 从该城府库扣费并生成同结构台账
```

AI 不得：

- 每月免费增加农业、商业和驻军。
- 在没有本地粮食或运输路线时免费维持大军。
- 让同一名武将在多座城市同时执行命令。
- 跳过建设时间或设施槽限制。
- 因为不展示界面而获得额外城务额度。

AI 势力性格只改变优先级，例如守成型更重视治安与城防、扩张型更愿意压低储备征兵，不改变底层规则。

## 14. 数值基线

以下为首轮实现与测试基线，后续可根据模拟调整，但不得改变“每城独立”的结构。

| 项目 | 基线 |
| --- | ---: |
| 每城基础城务额度 | 2/月 |
| 每城额度下限/上限 | 1/4 |
| 普通命令消耗 | 1 点本城城务 |
| 同一武将普通执行上限 | 1 次/月 |
| 太守委任额外处理 | 最多 1 次/月 |
| 设施队列 | 每城 1 条 |
| 城市主要定位 | 每城 1 个 |
| 定位切换过渡期 | 3 个月 |
| 新占城主要能力恢复 | 最少 3 个月 |
| 正常城市默认月报弹窗 | 0 个，汇总展示 |

### 14.1 推荐的压力目标

- 拥有 2 城时，玩家可以每月手动细管两城，完整操作不超过 2 分钟。
- 拥有 8 城时，使用方针后每月必须亲自处理的城市通常不超过 3 座。
- 拥有 16 城时，常规月份的重要内政提醒不超过 5 条。
- 同时拥有 10 城不代表势力只有 3 次内政；按基础值应具有 20 点分布在各城、不可跨城挪用的本地城务额度。

## 15. 关键流程示例

### 15.1 两城独立治理

刘备势力拥有徐州与下邳：

- 徐州本月城务为 3/3，玩家执行劝农、训练、修城，变为 0/3。
- 下邳本月城务仍为 2/2，不受徐州消耗影响。
- 下邳设置“富民通商”，太守糜竺在月末使用 2 点完成经商与巡查。
- 两城分别扣除自己的金粮，分别生成月报。

### 15.2 本城缺粮但中央有粮

- 前线城小沛粮食不足，势力中央府库仍有 20,000 粮。
- 玩家不能直接在小沛执行需要 3,000 粮的赈济。
- 界面提供“从下邳调拨”建议，显示道路、预计损耗、到达月份和势力政令消耗。
- 调拨完成后，小沛才能使用自己的城务额度执行赈济。

### 15.3 后期减负

- 玩家拥有 12 城，其中 8 城状态安定并已委任，2 城在建设，2 城位于前线。
- 月初总览只将两座前线城和一座缺粮城置顶。
- 玩家亲自处理 3 城，其余城市由各自太守使用各自额度执行。
- 月末只显示一张汇总报告，不要求逐城确认正常结果。

## 16. 开发切片

### 16.1 DOM-001：每城额度与全城总览

- 将界面中的“行动槽”统一改为“城务额度”。
- 确保额度逐城计算、逐城消耗、逐城重置。
- 新增己方城池总览，展示每城额度、状态、金粮和首要提醒。
- 修复自动治理只处理一座玩家城市的问题。

验收重点：一城额度耗尽不影响另一城执行命令。

### 16.2 DOM-002：城市府库闭环

- 月度产出进入城市府库。
- 本地命令、建设和驻军维护从城市府库扣除。
- 势力府库只通过上缴、调拨、战利品等明确路径变化。
- 增加逐城资源台账和资产守恒测试。

验收重点：任意一笔金粮能够追溯来源、去向和所属城市。

### 16.3 DOM-003：命令代价与武将执行

- 为八类命令补齐人口、粮食、军备、民心与训练联动。
- 同一武将每月只能执行一次普通城务。
- 命令执行前展示预计结果，执行后展示修正来源。
- 把平铺按钮改为生产、民生、军备、人才四组。

验收重点：征兵、训练、赈济不再只是扣中央金钱后加数值。

### 16.4 DOM-004：城市方针与太守委任

- 实现六种城市方针、储备底线和禁止项。
- 太守在月末使用本城剩余额度自动执行。
- 新增异常暂停、建议确认和汇总月报。
- 自动治理与手动治理使用同一命令处理器。

验收重点：8 城势力可以只亲自处理异常城市，其余城市仍按规则正常发展。

### 16.5 DOM-005：定位、调拨与跨系统联动

- 实现五种城市定位及其首个核心能力。
- 调拨加入势力政令、道路与简化运输损耗。
- 出征、驻军维护、占领整合与灾害接入本地府库。
- AI 全面改用玩家同规则。

验收重点：城市地理与定位能改变经营价值，偏远前线无法无条件共享全国资源。

## 18. 验收标准

### 18.1 功能验收

- [ ] 每座己方城市都有独立城务额度，任何城市的消耗不改变其他城市额度。
- [ ] 势力拥有多少城市，就有多少组不可跨城挪用的本地额度，不存在“全势力共用三次内政”。
- [ ] 城市普通命令和建设优先从本城府库扣费。
- [ ] 城市收入先进入本城府库，只有明确上缴或调拨后才进入势力府库。
- [ ] 玩家能从内政总览看到全部己方城市的状态、额度、金粮和异常。
- [ ] 玩家能逐城手动治理，也能为每城单独设置方针和储备底线。
- [ ] 自动治理使用本城武将、本城额度和本城资源，不产生免费收益。
- [ ] 徐州用完额度后，下邳仍可执行命令；测试覆盖至少 2 城和 10 城场景。
- [ ] 同一武将不能在同月、不同城市重复执行命令。
- [ ] AI 城市遵守与玩家相同的资源、额度、武将和建设规则。

### 18.2 体验验收

- [ ] 玩家进入内政页后，5 秒内能看出哪些城市需要处理。
- [ ] 单城页面不再同时平铺八个命令和所有设施操作。
- [ ] 每个命令执行前能看见所属城市、费用、预计收益与副作用。
- [ ] 8 城势力在委任后，普通月份不需要逐城重复点击。
- [ ] 正常月末只出现一张汇总月报，异常城市可直接跳转。
- [ ] “城市府库不足”和“势力府库不足”使用不同且带城名的提示。

### 18.3 数值与稳定性验收

- [ ] 连续模拟 60 回合后，不出现城市金粮无来源增长或同一收入重复入账。
- [ ] 转账、上缴和运输前后满足总资产守恒，损耗有独立台账记录。
- [ ] 低治安、缺粮、前线、新占和太守空缺城市会选择不同治理优先级。
- [ ] 城市数量增加会提高总产出和总执行能力，也会增加人员、维护、调拨与提醒压力。
- [ ] `./tools/run-godot.sh check`、`smoke`、`test` 全部通过，并新增每城额度与城市府库的自测。

## 19. 完成定义

本需求完成后，玩家应能清楚感受到：

- 自己管理的是徐州、下邳、小沛等一座座具体城市，而不是对整个势力按三次通用按钮。
- 每座城有自己的钱粮、问题、人才和发展方向。
- 前线城、粮仓、商埠与学府承担不同职责，城池不再只是换名字的同一组数值。
- 多城带来更多能力，但稳定城市能够交给太守，不会把游戏拖成逐城打卡。
- 内政的结果会自然进入战争、补给、人才、灾害与占领恢复，形成完整的长期经营循环。
