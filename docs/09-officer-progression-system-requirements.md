# 武将成长、特性与官职系统需求

> 文档版本：V1.0  
> 编写日期：2026-07-14  
> 状态：首版核心功能已实现（等级、成长、特性、特殊部队、官职、俸禄与 AI）  
> 适用工程：`godot/` 下的 Godot 4.7 .NET / C# 正式版本

> 实现基线：当前存档 schema v9；核心规则位于 `godot/scripts/OfficerProgressionSystem.cs`，并已接入内政、战斗、战略 AI、人才页和出征页。后续若扩展志向、承诺、派系与完整后天专长，继续按 `docs/01-requirements.md` 的 M17 后续切片实施。

## 1. 文档目的

本需求用于把现有武将从“固定五维加一个职责名称”扩展为可长期培养、可差异化任用、同时会产生财政成本的人才系统。

本次改造必须形成以下闭环：

```text
执行真实任务并形成履历
  → 获得阅历与功勋
  → 等级提升并按个人成长路线增加五维
  → 特性在合适的城务、职责、兵种或特殊部队上生效
  → 达到等级、功勋与履历要求后获得晋升资格
  → 官职带来条件属性和职务效率加成
  → 势力按月支付更高俸禄
  → 玩家在人才效果、忠诚承诺、编制名额与财政压力之间取舍
```

本文是武将等级、五维成长、特性、特殊部队适配、官职、俸禄和相关 AI 规则的专项需求来源。若与 `docs/01-requirements.md` 第 9、10、19.4、28 节中的较早概述冲突，以本文为准；战斗数值总上限继续服从 `docs/07-battle-system-requirements.md`，内政资源流继续服从 `docs/08-domestic-system-requirements.md`。

## 2. 当前实现审计

### 2.1 已有基础

当前武将已经具备：

- 统率、武力、智力、政治、魅力五维，范围 1–100。
- `Merit` 功勋、忠诚、健康、疲劳、所属势力、所在城池和出征状态。
- `Traits` 字符串列表，剧本中已有忠义、豪勇、谨慎、善谋、善政、野心、沉着、多疑、爱民、贪财十类标签。
- `Appointment`，当前保存君主、太守、军师、文官、武将和待命等职责。
- 战斗中的主将、副将职责，以及内政中的太守、执行武将和五维公式。
- M17 已定义履历、后天专长、志向、承诺、议政和军团协同方向。

### 2.2 当前缺口

| 模块 | 当前情况 | 问题 |
| --- | --- | --- |
| 等级 | 没有等级和阅历字段 | 武将参加多年治理或战争后五维不变，缺少长期成长反馈 |
| 五维 | 只有剧本初始值 | 无法区分初始能力、永久成长和临时加成 |
| 特性 | 只有字符串标签 | 没有统一触发条件、效果、品质、目标兵种、叠加规则和战报来源 |
| 名将 | 主要靠初始五维较高体现 | 名将与普通武将可能拥有完全相同的“豪勇、忠义”，特性辨识度不足 |
| 文臣 | “善政、善谋”等尚未进入完整命令公式 | 文臣在农业、商业、建设、人才、外交和委任方面缺少稳定价值 |
| 特殊部队 | 当前只有五个基础兵种 | 无法表达白马义从、陷阵营等精锐编制，也无法让专属将领产生适配加成 |
| 官职 | `Appointment` 同时承担职责名称 | 没有官阶、文武序列、晋升要求、编制名额与官职加成 |
| 俸禄 | 月结算文字提到官员维护，但未按武将结算 | 提拔没有长期成本，后期可以无条件让所有人身居高位 |
| AI | 不评估成长、特性或晋升 | 玩家和 AI 无法使用同一套培养与财政取舍 |

## 3. 设计目标与边界

### 3.1 核心目标

1. **成长来自履历**：战斗、治理、外交、情报和救援都能培养对应人才，但不能靠反复点击无代价刷级。
2. **等级有用但不抹平人物**：等级稳定提高五维，个人成长上限和路线仍保留名将、专才与普通武将的差异。
3. **特性改变适用场景**：特性优先改变具体命令、兵种、阶段或代价，不做无条件全局乘区。
4. **名将强在独特规则**：名将拥有更高品质或专属特性，但不获得隐藏的“名将全能力倍率”。
5. **文臣能够主导内政**：文臣特性主要作用于城市经营、委任、人才、外交、补给和行政，不强迫所有武将都以战斗升级。
6. **官职是收益与成本的选择**：晋升提供条件属性、职责效率、统兵或治理能力，同时永久提高月度俸禄并占用高阶名额。
7. **所有来源可解释**：任何有效属性、命令收益、战斗加成和月度俸禄都能展开查看来源。

### 3.2 明确不做

- 不做装备刷取、随机词条、转生、突破、升星或付费式养成。
- 不允许无限等级；首版等级上限固定为 20。
- 不允许玩家自由把所有成长点堆入单一五维，避免把任何人培养成同一模板。
- 不因一次升级随机降低属性，不使用读档刷随机成长。
- 不让官职永久修改人物的历史基础五维；离任后官职效果立即失效。
- 不把官职、实际职务和战场职责合并为同一字段。
- 不让一名武将仅凭特性“凭空召唤”特殊部队；特殊部队仍需满足势力、对应兵种与兵源条件，匹配特性只提供额外战斗加成。
- 不制作完整朝廷、爵位、封邑、世袭和后宫体系；本需求只处理势力内部官职与俸禄。

## 4. 核心概念与字段边界

### 4.1 四种容易混淆的身份必须分开

| 概念 | 示例 | 决定内容 | 是否可并存 |
| --- | --- | --- | --- |
| 实际职务 `Appointment` | 太守、军师、文官、武将、待命 | 当前在哪个岗位工作、可执行哪些命令 | 每人同时最多一个常设职务 |
| 官职序列 `OfficeTrack` | 文职、武职 | 晋升走哪条官阶表、主要加成方向 | 每人同时只能选择一条，允许付代价转序 |
| 官阶 `OfficeRank` | 0–7 阶 | 俸禄、高阶名额、条件属性与身份待遇 | 每人恰好一个官阶 |
| 战场职责 | 主将、前锋、军师、侧翼、后军、攻城督 | 某次战斗中影响哪个阶段和战斗队 | 只在当前军团或战斗中存在 |

太守是实际职务，不等于高官；一名低阶武将可以在人才紧缺时署理太守，但效率有限。一名高阶文官也可能处于待命状态，此时仍领取官阶俸禄，但不会获得太守职务效果。

### 4.2 五维的三层值

界面和公式必须区分：

```text
基础值 = 剧本当前时点的人物能力，不随任免变化
永久值 = min(100, 基础值 + 等级成长值 + 明确的永久事件值)
有效值 = clamp(永久值 + 当前官职条件加成 + 状态修正, 1, 110)
```

- 人物详情主数字显示永久值，旁边展示“基础 + 成长”。
- 官职、受伤、疲劳等只在当前任务预估中形成有效值，不写回永久值。
- 特性的百分比和规则效果独立显示，不折算进五维，避免重复计算。
- 现有所有直接读取 `Profile.Abilities` 的公式最终应改为统一能力查询，不允许战斗、AI 和 UI 各自拼接加成。

## 5. 等级与阅历

### 5.1 等级范围

- 等级范围：1–20。
- 等级代表处理军政事务的成熟度，不代表历史名气，也不直接等于年龄、官职或功勋。
- 等级只升不降；重伤、疲劳和衰老通过状态修正处理，不回退等级。
- 达到 20 级后不再获得阅历，但仍可获得功勋、履历、关系与事件结果。

### 5.2 累计阅历门槛

| 等级 | 累计阅历 | 等级 | 累计阅历 |
| ---: | ---: | ---: | ---: |
| 1 | 0 | 11 | 4,250 |
| 2 | 200 | 12 | 4,950 |
| 3 | 450 | 13 | 5,700 |
| 4 | 750 | 14 | 6,500 |
| 5 | 1,100 | 15 | 7,350 |
| 6 | 1,500 | 16 | 8,250 |
| 7 | 1,950 | 17 | 9,200 |
| 8 | 2,450 | 18 | 10,200 |
| 9 | 3,000 | 19 | 11,250 |
| 10 | 3,600 | 20 | 12,500 |

门槛是首轮平衡基线。修改门槛可以调整成长速度，但不得改成每次执行按钮立刻升级的短循环。

### 5.3 阅历来源

阅历只在任务正式结算时发放，同一件事不能按动画阶段、战报条目或 UI 操作重复发放。

| 履历来源 | 基础阅历 | 说明 |
| --- | ---: | --- |
| 完成普通城务 | 20 | 每月每名武将最多计一次普通城务阅历 |
| 完成建设、升级或城市整合节点 | 30 | 工程完成时发放，不按施工月重复发放 |
| 作为太守完成正常月结算 | 12 | 本月实际在任且城市未失控；不能与同月第二次署理重复 |
| 完成搜索、情报或外交行动 | 30–50 | 根据任务难度与结果等级确定 |
| 参与一次正式战斗 | 35 | 必须有正式职责并进入贡献记录 |
| 担任主将完成战斗 | 额外 25 | 胜负都可获得，溃败且严重失职时不发额外值 |
| 胜利或完成关键战术目标 | 额外 20–60 | 由战斗结果和个人贡献决定 |
| 完成救援、断后、守城或战役目标 | 40–100 | 每个结构化目标只结算一次 |
| 完成季度方略或重要承诺 | 60–120 | 只发给实际建议者或责任人 |
| 重大事件履历 | 20–150 | 必须在事件定义中明确，不允许临时代码写死 |

### 5.4 防刷与一致性规则

- 打开页面、任命后立刻撤销、取消建设、军团未接战和失败校验不提供阅历。
- AI 势力、玩家手动城务和玩家单城委任使用同一阅历值与成长规则。
- 同类普通城务连续执行仍提供基础阅历，但只有达到真实结果门槛时才形成“屯田、治水、商贸”等履历进度。
- 单名武将每月普通阅历建议上限为 200；重大事件和一场已结束的大战可以突破，但必须在记录中列明来源。
- 阅历不消耗功勋，功勋也不自动换算为阅历。两者分别保存。
- 同一存档、命令和随机种子必须产生相同阅历与升级结果。

### 5.5 升级生效时点

- 城务、外交和战斗先用行动开始时的能力快照完成本次结算。
- 任务结束后发放阅历；达到门槛时记录升级与成长。
- 新永久属性从该任务结束后生效，不反向修改已经完成的损耗和收益。
- 月末批量任务在全部同阶段结算完成后统一升级，避免势力处理顺序造成先行动者占优。

## 6. 等级对属性的影响

### 6.1 个人成长方案

每名武将拥有确定性的 `GrowthPlan`，规定 2–20 级每一级增加哪项能力。成长方案由人物定位和潜力制定，不在升级时随机抽取。

- 每次升级至少获得 1 点永久五维成长。
- 5、10、15、20 级为成长节点，额外获得 1 点，因此从 1 级升至 20 级最多获得 23 点总成长。
- 单项普通成长上限为初始值基础上的 +8；明确标记为年轻潜才的单项上限可以达到 +12。
- 任一永久五维仍以 100 为上限。
- 预定属性已达个人上限时，按成长方案的次优先级转移；全部达到上限时转为 50 功勋，并明确提示。

### 6.2 默认成长原型

未配置个人方案时，按原型生成固定方案：

| 原型 | 成长主次 | 典型人物定位 |
| --- | --- | --- |
| 统帅 | 统率 ＞ 智力 ＞ 魅力 ＞ 武力 ＞ 政治 | 主将、守将、军团组织者 |
| 猛将 | 武力 ＞ 统率 ＞ 魅力 ＞ 智力 ＞ 政治 | 前锋、追击、护阵 |
| 谋士 | 智力 ＞ 政治 ＞ 统率 ＞ 魅力 ＞ 武力 | 军师、计策、情报 |
| 能吏 | 政治 ＞ 智力 ＞ 魅力 ＞ 统率 ＞ 武力 | 太守、建设、农业 |
| 名士 | 魅力 ＞ 政治 ＞ 智力 ＞ 统率 ＞ 武力 | 招募、赈济、外交 |
| 全才 | 两项主能力交替，其余均衡 | 君主、综合型核心人才 |

正式剧本中的名将和关键文臣应使用人工配置的个人方案；默认原型只作为普通人物与数据缺失保护。

### 6.3 成长展示

升级提示必须展示：

```text
赵云升至 8 级
统率 92 → 93（基础 91，成长 +2）
下一官阶“中郎将”仍需功勋 1,200：当前 1,080
```

若没有五维变化而转换成功勋，也必须说明属性已经达到个人上限，不能静默吞掉成长。

## 7. 功勋、等级与履历的分工

| 系统 | 回答的问题 | 主要来源 | 主要用途 |
| --- | --- | --- | --- |
| 等级/阅历 | 做事是否更成熟 | 各类有效任务 | 永久五维成长、官阶等级门槛 |
| 功勋 | 为当前势力立了多少功 | 战果、治理成果、承诺与方略 | 晋升、派系声望、议政资格 |
| 履历 | 具体擅长做过什么 | 结构化任务计数与关键结果 | 后天专长、晋升特殊条件、人物列传 |
| 忠诚/信任 | 是否愿意继续效力 | 任用、公平、承诺、关系 | 离职、提案、策反和危机支持 |

- 不允许只靠等级获得最高官职；还必须满足功勋、履历和名额。
- 不允许只靠高初始功勋直接获得全部成长属性。
- 后天专长继续按 M17 的履历条件获得，每人最多两个，不改为“每五级自动抽技能”。

## 8. 特性体系

### 8.1 特性分类

| 分类 | 作用重点 | 示例 |
| --- | --- | --- |
| 性格特性 | 忠诚、提案、风险偏好、关系与少量条件效果 | 忠义、谨慎、野心、多疑、贪财 |
| 军事特性 | 战斗阶段、职责、基础兵种、战术与损耗 | 豪勇、善谋、守城、骑将、强弩 |
| 内政特性 | 城务、建设、委任、行政、补给与人才 | 善政、爱民、屯田、营造、富商、礼贤 |
| 名将特性 | 人物专属或极少数共享的规则能力 | 白马将军、陷阵之志、江东英略 |
| 后天专长 | 由履历获得的可选能力 | 屯田能手、守城宿将、纵横家 |

一项特性可以同时属于一个主分类和若干标签，但必须有唯一 `TraitId`。界面名称相同但规则不同的特性不得复用同一 ID。

### 8.2 特性品质

| 品质 | 定位 | 常见强度 | 规则数量 |
| --- | --- | ---: | ---: |
| 普通 | 明确的小场景优势或性格倾向 | 2%–4% 或一个小规则 | 1 个主要效果 |
| 稀有 | 构成稳定专长，可影响一类命令或兵种 | 4%–6% | 1 个主要效果，可带 1 个小代价/条件 |
| 传奇 | 名将辨识度与特殊部队适配 | 单项仍不超过 6% | 1 个主要数值效果 + 1 个规则效果 |

传奇特性“更强”主要体现在更精确的适用对象、规则能力和组合价值，不通过突破战斗系统总上限实现。例如“陷阵之志”可以让陷阵营在低士气时延后进入溃散，同时提供不超过 6% 的接战加成，而不是简单给予全军 +20% 攻击。

### 8.3 名将与普通武将的配置标准

人物资料新增 `FameTier`：普通、知名、名将。它只决定初始特性内容预算与展示，不作为隐藏倍率。

| 人物层级 | 初始特性建议 | 约束 |
| --- | --- | --- |
| 普通 | 1–2 个普通特性 | 至少一个能在其主要岗位生效 |
| 知名 | 1 个稀有 + 1–2 个普通特性 | 稀有特性需体现兵种、战术或治理方向 |
| 名将 | 1 个传奇专属 + 1 个稀有/普通特性 | 专属特性不能被普通随机成长获得 |

- 名将不自动获得额外经验倍率、全属性倍率或俸禄减免。
- 普通武将通过履历可以成为可靠专才，但不能复制人物专属传奇特性。
- “名将”不是永远更适合所有岗位；顶级猛将治理城市仍可能弱于普通能吏。

### 8.4 数据要求

每个特性定义至少包含：

```text
TraitDefinition
├── Id / Name / Description
├── Category / Quality / Tags
├── Trigger                 # 城务、战斗类型、阶段、职责、地形等
├── TargetTags              # 基础兵种、特殊部队、城市定位或命令
├── Effects                 # 数值或规则效果列表
├── RequiredAssignment      # 主将、副将职责、太守、执行者等
├── MinCompositionShare     # 兵种占比门槛
├── StackGroup / StackRule / Cap
├── ReportText
└── AiValueHints
```

通用代码不得按武将 ID 判断关羽、吕布、高顺或公孙瓒；人物差异必须来自特性和特殊部队配置表。

## 9. 军事特性与特殊部队

### 9.1 特殊部队定义

特殊部队不是第六种基础兵种，而是继承步兵、枪兵、弓兵、骑兵或攻城器械之一，并附加标签、招募条件、成本和少量规则差异的精锐编制。

```text
SpecialTroopDefinition
├── Id / Name
├── BaseTroopType
├── Tags
```

| 示例特殊部队 | 基础兵种 | 关键标签 | 适配方向 |
| --- | --- | --- | --- |

### 9.2 特性生效条件

特殊部队加成同时满足以下条件才生效：

1. 军团真实编制中存在目标特殊部队。
2. 该特殊部队不少于军团总兵力的 20%，且不少于 500 人。
3. 特性持有者在本军团有正式职责，并负责该兵种、对应阵位或相关战术。
4. 特性定义的地形、阶段、阵型或状态条件满足。
5. 武将未重伤、被俘、离队或失去指挥能力。

只带极少量特殊部队不能获得完整全军加成。未满足条件时，战前界面把特性置灰并显示具体原因，例如“白马义从占比 12%，未达到 20%”。

### 9.3 叠加与上限

- 同一 `StackGroup` 中最高品质效果按 100% 生效，第二来源按 50% 生效，其余不再叠加。
- 单个战斗特性的同一数值效果通常为 2%–6%。
- 同一战斗阶段全部武将特性的综合影响仍不得超过 15%。
- 全部武将因素对全军有效战力的综合影响仍服从 0.85–1.25 限制。
- 特性、官职和基础兵种修正分别记录来源，再由统一修正器应用；禁止先改属性又重复乘一次同名效果。

## 10. 文臣与内政特性

### 10.1 设计原则

- 文臣特性一般加成内政，但不统一写成“全部内政 +X%”。
- 每项特性应对应一种命令、城市定位、太守职责、委任策略或资源代价。
- 文臣执行城务、担任太守、军师、外交官、后军或攻城督都可以获得阅历和功勋。
- 政治不是唯一有效能力；智力、魅力以及少量统率仍按现有内政专项规则参与。

### 10.2 首批特性效果基线

| 特性 | 品质 | 触发条件 | 推荐效果 | 代价/边界 |
| --- | --- | --- | --- | --- |
| 善政 | 普通 | 执行农业、商业或担任太守 | 对当前命令最终效果 +4% | 不同时提高所有城市 |
| 爱民 | 普通 | 赈济、整顿治安、征兵 | 赈济/巡查效果 +4%；征兵民心损失 -1 | 每次命令最多减少 1 点民心损失 |
| 善谋 | 普通 | 情报、寻访、工程规划、军师 | 对应评分 +4% | 不直接增加税粮 |
| 沉着 | 普通 | 城市危机、灾害、被围 | 危机类命令波动缩小，失败下限提高 | 正常繁荣期不提供收益 |
| 屯田 | 稀有 | 执行农业；城市有灌溉渠或军镇驻军 | 农业命令 +6%，本城驻军粮耗 -3% | 只作用当前城，粮耗减免有总上限 |
| 营造 | 稀有 | 建设或修复设施 | 工期达到条件时缩短 1 月，最低仍为 1 月 | 同一工程只触发一次 |
| 富商 | 稀有 | 商业命令或商埠治理 | 商业效果 +6%，贸易损耗 -2% | 灾害或道路中断时贸易效果失效 |
| 礼贤 | 稀有 | 搜索、举荐、招募 | 人才发现评分 +6%，招募解释增加举荐项 | 不保证必定发现或加入 |
| 能吏 | 稀有 | 太守委任治理 | 本城自动治理候选评分更准确，异常误判减少 | 不增加本城行动槽 |
| 名士领袖 | 传奇 | 议政、招募、学府治理 | 解锁一次额外高质量提案候选；相关评分 +6% | 每季度至多触发一次 |

具体值可通过模拟调整，但每项效果必须保留适用范围和上限。

### 10.3 内政叠加规则

- 一条城务由执行者特性完整生效，太守协同特性只按 50% 生效，除非特性明确写为太守光环。
- 同一命令全部特性乘算修正建议限制在 0.85–1.15。
- 固定值变化先结算，百分比修正后结算，最后应用城市属性上下限。
- 预估与正式结算必须调用同一规则；确定性波动只在所有可见修正之后应用。

## 11. 官职与官阶

### 11.1 文武双序列

首版提供文职与武职两条官阶序列。名称是当前群雄政权下的游戏化称谓，后续可以按势力政体替换显示名，但内部阶级和数值规则保持一致。

| 阶 | 文职示例 | 武职示例 | 等级要求 | 功勋要求 | 月俸 | 条件属性加成 |
| ---: | --- | --- | ---: | ---: | ---: | --- |
| 0 | 白身 | 白身 | 1 | 0 | 0 | 无 |
| 1 | 从事 | 军司马 | 2 | 200 | 10 | 文：政治 +1；武：统率 +1 |
| 2 | 功曹 | 校尉 | 4 | 600 | 22 | 文：政治 +2；武：统率 +2 |
| 3 | 别驾 | 中郎将 | 6 | 1,200 | 40 | 文：政治 +2、智力 +1；武：统率 +2、武力 +1 |
| 4 | 治中 | 偏将军 | 9 | 2,200 | 65 | 文：政治 +3、魅力 +1；武：统率 +3、武力 +1 |
| 5 | 尚书 | 杂号将军 | 12 | 3,600 | 100 | 文：政治 +4、智力 +2；武：统率 +4、武力 +2 |
| 6 | 九卿 | 重号将军 | 15 | 5,500 | 150 | 文：政治 +5、智力 +3；武：统率 +5、武力 +3 |
| 7 | 三公 | 大将军 | 18 | 8,000 | 220 | 文：政治 +6、智力 +4；武：统率 +6、武力 +4 |

上述属性只在匹配任务中形成有效值：

- 文职政治、智力、魅力加成用于城务、太守治理、行政、人才、情报、外交、后勤与议政。
- 武职统率、武力加成用于出征、守城、训练、征兵整编和正式战斗职责。
- 文官被临时安排为前锋时不会把文职政治加成转换成武力；武官执行商业命令也不会获得武职统率加成。
- 有效值上限为 110，官职不写回永久五维。

### 11.2 官阶附加能力

为避免官职只有裸属性，高阶官职提供一个小型规则能力：

| 官阶 | 文职附加能力 | 武职附加能力 |
| ---: | --- | --- |
| 1–2 | 无额外规则，只提供条件属性 | 无额外规则，只提供条件属性 |
| 3 | 担任太守时自动治理候选预估误差降低 | 担任主将时建议统兵容量 +2% |
| 4 | 担任太守时本城行政能力 +1 | 负责战斗队的撤退秩序 +3% |
| 5 | 可主持区域委任；区域行政损耗 -2% | 可建立高级军团编制；特殊部队维护 -3% |
| 6 | 每季度可额外提交 1 个议政候选，但不增加玩家可采纳数量 | 军团可多保存 1 套备用阵型，不增加参战兵种上限 |
| 7 | 全势力只能生效 1 人；重大行政危机可提供一次复核 | 全势力只能生效 1 人；大型战役指挥协同上限 +3% |

附加能力只在任职和条件满足时生效。相同官阶能力不跨多人无限叠加。

### 11.3 朝堂官职树

朝堂职位与文武官阶并行，不替代成长官阶。主公之下设置三系主官，每系下辖三席；这是采用汉魏官名的游戏化组织树，不宣称各官名在史实中存在固定的现代上下级关系。

| 主官 | 下辖席位 | 适用序列 | 势力效果方向 | 月俸津贴 |
| --- | --- | --- | --- | ---: |
| 大将军 | 前将军、左将军、右将军 | 武职 | 主官强化全军战力、训练和士气；属官分别偏向进攻、攻坚和军心 | 45–100 |
| 丞相 | 尚书令、治中从事、别驾从事 | 文职 | 主官强化全势力钱粮与城务；属官分别偏向政令、治理和商贸运输 | 40–100 |
| 军师将军 | 中军师、护军、军司马 | 文职/武职 | 主官强化战术计策；属官分别偏向谋划、武将体力军心和军务后勤 | 40–90 |

- 同一席位只能有一名现任；已有现任时必须先卸任，不能直接替换。
- 同一武将只能担任一个朝堂职位；转任前必须先卸任原职位。
- 主官至少需要 3 阶官阶，属官至少需要 2 阶，并校验文武序列与在任状态。
- 职位津贴进入真实月俸、欠俸、府库支付与财政风险计算；朝堂职位不再给任职者固定裸属性，而是将职位权力、任职者对应能力和个人特性合成为势力效果。
- 同一职位由不同武将担任时，能力折算和特性转化结果必须不同；普通特性提供方向性加成，传奇名将特性提供更强且可辨识的特殊效果，例如全军战力、战术效果、武将体力、士气、钱粮收入或城务效率。
- 三位主官产生完整的全势力光环；九席属官只强化各自专业领域，个人特性转化强度按主官的 55% 计算。各类势力加成分别设置总上限，避免十二席叠满后数值失控。
- 人才府必须在选择候选后、正式任命前显示动态结果，至少包括势力加成、参与转化的特性、任后总俸和任职门槛；不能只显示职位的固定说明。

### 11.4 晋升条件

晋升必须同时满足：

1. 达到目标官阶的等级要求。
2. 达到目标官阶的功勋要求。
3. 目标序列与主要履历相符，3 阶以上至少完成一项对应履历门槛。
4. 势力拥有可用高阶名额。
5. 晋升后预计俸禄不突破玩家设置的财政警戒线，或由玩家明确确认承担风险。
6. 武将不是俘虏、在野、死亡或正在叛逃事件中。

履历门槛示例：

- 文职 3 阶：完成 6 次有效城务、一次建设完成或一次外交/人才成果。
- 文职 5 阶：治理一城连续 6 月稳定，或完成一项季度方略。
- 武职 3 阶：参加 3 次正式战斗或完成一次守城。
- 武职 5 阶：作为主将获胜、完成救援/断后或形成显著战役贡献。
- 6–7 阶必须有重大履历，不能只靠重复普通命令累计。

### 11.5 高阶编制名额

| 官阶 | 势力名额基线 |
| ---: | --- |
| 0–2 | 不限 |
| 3 | 每座己方城市最多支持 2 人 |
| 4 | `1 + floor(城池数 / 2)` 人 |
| 5 | `1 + floor(城池数 / 4)` 人 |
| 6 | `floor(城池数 / 6)` 人，至少控制 6 城才解锁首名 |
| 7 | 文武合计 1 人，至少控制 12 城且威望达标 |

- 君主不占用普通官阶名额，也不从官阶表领取俸禄；君主开支由势力维护另行结算。
- 名额减少时不立即强制降职，标记为“超编”；超编官员仍领俸禄，但新增晋升被禁止，且行政负担增加。
- 高阶名额由城池数与威望共同校验，防止刚占空城立刻批量封官。

### 11.6 转序、降职与离任

- 文武转序保留官阶数值，但需要 1 个月适应期、消耗一次势力政令，适应期内官职加成减半。
- 4 阶以上转序还需相应履历，否则最多先转为 3 阶。
- 主动降职不返还已支付俸禄或威望成本，并产生至少 3 个月再次晋升冷却。
- 因财政危机降职必须提前提示；涉及招募承诺或志向时进入承诺违约判断。
- 撤销太守职务不等于降低官阶；两种操作必须分开确认。

## 12. 俸禄与财政

### 12.1 支付责任

- 所有仍属本势力且应领取俸禄的武将统一由势力府库支付；所在城市和是否随军不改变支付账户。
- 驻城、在途、出征和朝堂任职状态只影响人物可执行的行动，不建立城市级俸禄账户。
- 被俘、在野和死亡人物不领取本势力俸禄；重伤休养但仍保留官职者正常领取。
- 待命武将只要仍有官阶和所属势力，仍领取完整俸禄，防止通过闲置逃避成本。
- 同一武将每月只生成一笔应付俸禄，不能由城市与势力重复支付。

### 12.2 月度结算顺序

俸禄接入内政专项的固定顺序：

```text
1. 各城产出直接汇入所属势力府库
2. 势力府库支付各城驻军与设施维护
3. 势力府库统一支付本势力全部应付俸禄
4. 执行方针委任城务与非玩家势力 AI 行动
5. 汇总势力俸禄、欠俸与财政预警
```

金粮只有势力府库一个运行时账户，不提供城市俸禄保底拨款、上缴或调拨。

### 12.3 欠俸

资金不足时不允许把金钱扣成负数。按应付总额计算本月实付比例，并保存个人欠俸月份：

| 实付比例 | 当月结果 |
| ---: | --- |
| 100% | 正常，无负面 |
| 75%–99% | 记录欠俸并预警；首次不直接扣忠诚 |
| 50%–74% | 连续第二月时忠诚 -1，贪财/野心人物额外增加不满 |
| 1%–49% | 忠诚 -1；连续两月再 -1，并可能提出补发请求 |
| 0% | 忠诚 -2；连续三月可触发请辞、倒向派系或被策反风险 |

- 忠义人物不会免疫欠俸，只是负面事件权重较低。
- 补发欠俸需要明确支出；补发后清除对应记录并改善信任。
- 不允许单月欠俸直接导致无预警叛逃，继续遵守 M17 的提前预警原则。

### 12.4 财政信息

晋升确认页必须同时显示：

```text
当前月俸：40
晋升后月俸：65（+25/月）
势力当前总俸禄：486 → 511
近三月平均金收入：2,840
俸禄占比：17.1% → 18.0%
晋升后仍高于安全线：是
```

势力月报按城市、军团/在途和中央三类汇总俸禄，不要求玩家逐人查看，但必须能下钻到个人。

## 13. 特性、等级与官职的统一计算顺序

所有内政、战斗、外交和 AI 预估使用以下顺序：

```text
1. 读取人物基础五维
2. 加入等级永久成长，得到永久值（上限 100）
3. 根据任务类型加入匹配官职条件属性
4. 应用健康、疲劳、受伤、围困等状态修正，得到有效值（上限 110）
5. 将有效值代入原有职责或命令公式
6. 应用特性、特殊部队、设施、地形、关系和协同等条件修正
7. 应用各系统修正上限
8. 应用确定性波动
9. 输出结果与来源列表
```

来源列表至少区分“永久成长、官职、特性、特殊部队、状态、设施/地形”，禁止只显示一个无法追溯的总倍率。

## 14. 与现有系统的连接

### 14.1 战斗

- 主将统兵容量、职责评分和战斗队贡献读取有效五维。
- 特性只作用于定义的战斗阶段、职责和目标战斗队。
- 战前预估展示当前会生效的前三项人物因素；战后贡献列出实际触发次数和效果。
- 战斗结束后统一发放阅历、功勋与履历，不按动画帧发放。

### 14.2 内政

- 城务能力读取永久值和匹配文职官阶加成。
- 城务预估列出执行者特性、太守协同、设施和城市状态。
- 太守月报记录治理阅历、功勋、俸禄和是否达到后天专长条件。

### 14.3 人才、忠诚与承诺

- 招募时承诺的具体官职记录目标序列、最低官阶和兑现期限。
- 提前晋升可以增加信任，但不直接购买永久忠诚。
- 长期越级任用、明显功高不赏或频繁降职会影响志向和派系。
- 低等级名将仍保留强特性，但可能因经验不足无法担任最高官阶。

### 14.4 敌对势力 AI 与单城委任

- AI 与玩家使用相同阅历、成长、晋升、名额、俸禄和欠俸规则。
- AI 必须把特性适配纳入主将、副将、太守、城务执行者和特殊部队负责人选择。
- 玩家单城委任只允许 AI 选择该城普通城务执行者，不允许自动任命、晋升、外交或军事决策。

## 15. AI 决策规则

### 15.1 任用评分

AI 选择岗位时至少考虑：

```text
岗位效用 = 匹配职责的有效五维
         + 可触发特性价值
         + 特殊部队适配价值
         + 当前履历培养价值
         + 忠诚/承诺紧迫度
         - 调动时间与空缺风险
         - 疲劳和伤病风险
```

AI 不得只按等级或总五维排序。

### 15.2 晋升评分

```text
晋升效用 = 岗位增益价值
         + 履行官职承诺价值
         + 忠诚与派系稳定价值
         - 新增月俸 × 财政压力权重
         - 高阶名额机会成本
         - 超编与行政压力
```

- 常规情况下，晋升后总俸禄不应超过近三月平均金收入的 25%。
- 战时、继承危机或即将违背重要承诺时可以突破，但必须在 AI 审计中说明。
- AI 不因武将刚达到门槛就立即晋升；至少比较三名候选和“不晋升”。
- 财政危机优先暂停晋升、调整待命高官和补发欠俸，不应每月反复升降同一人。

## 16. 界面需求

### 16.1 人才总览

每名武将至少展示：

- 肖像、姓名、等级和当前阅历进度。
- 永久五维；存在官职或状态修正时用可展开标记显示。
- 当前职务、官职名称、官阶、月俸和所在位置。
- 最多三个核心特性图标，其余可展开。
- “可晋升、承诺将到期、欠俸、可获得专长”等状态。

支持按等级、官阶、月俸、特性分类、特殊部队适配和可晋升状态筛选。

### 16.2 武将详情

详情页分为四组：

1. **能力与成长**：基础值、永久成长、个人上限、下一级成长预览和阅历来源。
2. **特性与适配**：品质、触发条件、当前可用状态、适配兵种/特殊部队和叠加组。
3. **任职与待遇**：实际职务、文武序列、官阶、俸禄、晋升条件、名额与财政影响。
4. **履历与关系**：功勋来源、战斗/治理记录、后天专长进度、志向、承诺和欠俸历史。

### 16.3 晋升确认

晋升前必须显示：

- 晋升前后官名、官阶和月俸。
- 新增条件属性和附加能力，哪些当前岗位能生效。
- 等级、功勋、履历、名额和财政条件。
- 对承诺、忠诚、派系和其他候选人的已知影响。
- 势力近三月收入、当前总俸禄和晋升后占比。

### 16.4 战斗与内政反馈

- 战前选将时直接标记“特性已生效”“缺少目标兵种”“特殊部队占比不足”或“职责不匹配”。
- 城务执行前显示本次使用的有效能力，例如“政治 92 = 永久 88 + 尚书 4”。
- 战报和月报只默认展示影响最大的三项，允许展开全部来源。
- 不能只用颜色区分普通、稀有和传奇，必须同时显示文字或图形边框名称。

### 17.4 Schema v5 迁移

- 正式剧本应为 112 名武将人工配置初始等级；旧运行存档缺少等级时，可按年龄、功勋和有效履历生成 1–12 级兼容值，但不得反向追加成长属性。

- 现有 `Traits` 名称映射为稳定 `TraitId`；无法映射时拒绝静默丢弃，输出武将 ID 和原值。
- 现有 `Appointment` 继续作为实际职务，不得迁移成官阶。
- 君主初始化为特殊君主身份；其他武将按当前职责、功勋和人物配置初始化文武序列与官阶。
- 迁移前后金粮、五维、所属、位置、状态和功勋必须保持；新增俸禄从迁移后的下一个完整月份开始结算，避免读档瞬间重复扣款。

## 18. 数值平衡约束

- 1–20 级总永久成长最多 23 点，任一五维永久上限 100。
- 官职条件属性使有效值最高达到 110，不写回永久值。
- 单个战斗特性通常影响 2%–6%，同阶段全部武将特性不超过 15%。
- 单条城务全部特性乘算修正建议限制在 0.85–1.15。
- 特性不应同时无条件提高产出、降低成本、增加行动槽并提高成功率；传奇特性也要有明确场景。
- 特殊部队本身与名将特性组合后，单支同规模军团不能长期稳定超过普通精锐军团 25% 的有效战力。
- 常规势力总俸禄目标为近三月平均金收入的 10%–20%，25% 进入预警，35% 以上视为严重财政风险。
- 高阶官职提供可感知优势，但不应让一个 7 阶官员替代两到三名合适的中阶执行者。

## 19. 开发切片

### 19.1 OFF-001：等级与统一能力查询

- 新增等级、阅历、成长方案、成长值和潜力字段。
- 建立统一永久值/有效值查询服务。
- 接入城务、战斗、选将预估、AI 和人物详情。
- 完成升级时点、防重复发放和确定性测试。

验收：同一武将完成任务后能升级并按方案成长；读档、预估、正式结算和 AI 读取同一能力值。

### 19.2 OFF-002：数据驱动特性

- 将现有十类字符串特性迁移为定义表。
- 先落地 5 个军事特性、5 个内政特性和完整叠加/来源规则。
- 接入城务预估、战前选将、战报与月报。

验收：每个已落地特性至少有一个可自动测试的生效场景和一个不生效场景。

### 19.3 OFF-003：名将与特殊部队切片

- 录入人物层级和首批 8–12 个名将专属特性。
- 落地 3–5 个特殊部队定义、招募条件、编制保存和维护成本。
- 完成占比、职责、阶段、叠加和战报规则。

验收：同一名将带适配与不适配部队时产生明确差异；普通武将带同一部队仍可作战但没有专属规则。

### 19.4 OFF-004：官阶、晋升与俸禄

- 保留实际职务，新增文武序列、0–7 官阶、名额和晋升条件。
- 接入条件属性、附加能力、城市/势力工资责任、欠俸和台账。
- 完成人才总览、晋升确认和财政预警。

验收：晋升立即改变匹配任务预估，下月按唯一支付方增加俸禄；离任不等于降职，降职后加成消失。

### 19.5 OFF-005：M17 履历、专长、承诺与 AI

- 将等级阅历与现有功勋履历分开。
- 接入后天专长、官职承诺、志向、议政和派系。
- AI 使用相同的任用、晋升、财政和欠俸规则。
- 增加 60/120 回合长期压力验证，并确认单城委任不会越权为玩家执行任命、外交或军事决策。

验收：玩家与 AI 均能培养、晋升和负担武将，不产生免费成长、无限封官或负金。

## 20. 验收标准

### 20.1 等级与属性

- [ ] 所有武将具有 1–20 级、当前阅历和下一等级门槛。
- [ ] 有效任务只结算一次阅历，无成本操作和取消操作不提供阅历。
- [ ] 升级按确定性个人方案增加五维，永久值不超过 100。
- [ ] 官职和状态只改变有效值，不污染基础值与永久成长值。
- [ ] 相同存档、命令和种子产生相同升级与成长结果。

### 20.2 特性与名将

- [ ] 所有正式特性拥有稳定 ID、品质、触发条件、效果、叠加组和说明文本。
- [ ] 名将至少有一个具辨识度的稀有或传奇特性，普通武将不通过随机成长复制专属特性。
- [ ] 文臣至少有一个能在内政、人才、外交或行政中稳定生效的特性。
- [ ] 特性条件不满足时不生效，界面能说明原因。
- [ ] 战斗特性遵守单项与阶段总上限，内政特性遵守单命令上限。

### 20.3 特殊部队

- [ ] 特殊部队继承一个基础兵种并保存自身标签、人数、成本和维护。
- [ ] 适配特性只有在真实编制、最低人数、占比和职责条件都满足时生效。
- [ ] 一名武将不能让同一特殊部队在两个军团同时获得加成。
- [ ] 战前预估与战报使用同一适配来源和数值。

### 20.4 官职与俸禄

- [ ] 实际职务、文武序列、官阶和战场职责在数据与界面中明确分离。
- [ ] 晋升校验等级、功勋、履历、名额和财政，并显示新增月俸。
- [ ] 官职加成只在匹配任务中生效，离任/转序/降职规则符合定义。
- [ ] 每名武将每月最多生成一笔俸禄，城市和势力不能重复支付。
- [ ] 资金不足不会产生负金，欠俸具有预警、累积后果与补发闭环。
- [ ] AI 遵守相同晋升名额、俸禄和财政安全线。

### 20.5 存档与稳定性

- [ ] v4→v5 迁移不降低旧武将五维，不丢失功勋、特性、所属、位置与状态。
- [ ] 新增俸禄从迁移后的下一个完整月份开始扣除。
- [ ] 60–120 回合模拟中无重复阅历、重复支付、无限晋升、负资源和特性越界叠加。
- [ ] 敌对势力 AI 与单城委任使用相同成长和城务规则，且单城委任不会越权代替玩家作出势力级决策。

## 21. 关键流程示例

### 21.1 文臣成长与晋升

陈登连续完成农业治理、人才寻访和一次城市整合：

```text
农业城务 +20 阅历，屯田履历 +1
人才寻访 +40 阅历，识人履历 +1
整合节点 +30 阅历，治理履历 +1
累计达到 3,000 阅历，升至 9 级
按个人成长方案：政治 +1、智力 +1（成长节点额外点）
达到治中文职等级门槛，但功勋仍差 180
```

之后完成季度方略，功勋达到要求。玩家晋升其为治中，政治有效值在太守治理任务中增加 3、魅力增加 1，月俸从 40 增至 65。晋升页同时提示势力总俸禄每月增加 25。

### 21.2 名将与特殊部队

高顺拥有传奇特性“陷阵之志”，军团编有 2,400 陷阵营、7,600 其他步枪兵，高顺被任命为前锋并负责陷阵营：

- 陷阵营占军团 24%，超过 20% 门槛。
- 接战阶段获得定义内的 6% 局部修正。
- 低士气时触发一次“严整不溃”规则，延后溃散判断。
- 加成只作用于高顺负责的陷阵营战斗队，不提高全军远程或攻城能力。

若高顺没有随军、被安排为后军但未负责陷阵营，或陷阵营只占 12%，专属特性不生效并在战前明确置灰。

### 21.3 高官财政压力

一个六城势力有 18 名武将，其中 4 人被提拔到 4–5 阶。虽然岗位预估提高，但总俸禄达到近三月平均金收入的 31%。系统在下一次晋升时提示高风险；若仍晋升并遭遇灾害，可能出现欠俸。

玩家可以选择：

- 暂停其他支出，优先补发俸禄。
- 将一名长期待命高官降职，承担承诺与关系后果。
- 发展各城商业与市集，使月度收入直接汇入势力府库，长期提高可负担规模。

官职因此成为人才布局与财政经营的持续取舍，而不是一次点击后的永久免费属性。

## 22. 完成定义

本需求完成后，玩家应能清楚感受到：

- 同一名武将在多年治理与征战后确实会成长，但成长路线仍符合人物定位。
- 名将的专属特性比普通标签更有辨识度，尤其能在适配特殊部队时改变局部战法。
- 文臣通过内政、人才、外交和行政持续成长，不需要被迫上战场刷经验。
- 提拔一名武将既是认可和能力提升，也是可持续的月度财政承诺。
- 等级、五维、特性、官职、俸禄、忠诚和履历互相连接，但每一项仍有清楚的字段边界和可追溯来源。
