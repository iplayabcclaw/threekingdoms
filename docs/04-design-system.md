# 视觉与交互设计规范

## 1. 视觉定位

关键词：**山河沙盘、汉末质感、克制华丽、清晰数据、人在局中**。

画面采用写实历史氛围与水墨纸张质感结合的方向。精致感来自材质、光影、层级、留白和微动效，而不是满屏金边、强饱和红色或复杂装饰。

## 2. 表现层结构

每个主要场景分四层：

1. **氛围背景层**：生成式场景图、雾、光照和远景。
2. **世界信息层**：城池、路线、军团、旗帜和状态特效。
3. **操作界面层**：卡片、按钮、筛选、资源和表单。
4. **反馈层**：提示、战报、事件、数值变化和转场。

背景层永远不承载正式数据。即使更换整张图，游戏仍应完整可操作。

## 3. 色彩系统

### 3.1 中性色

| 令牌 | 色值 | 用途 |
| --- | --- | --- |
| `--ink-900` | `#1F2A25` | 转场遮罩、深墨描边 |
| `--ink-800` | `#202B26` | 主要文字、标题 |
| `--ink-600` | `#46534D` | 次要文字、说明 |
| `--paper-100` | `#F2ECDF` | 普通面板、卡片 |
| `--paper-50` | `#FFFAF0` | 顶栏、浮层、输入框 |
| `--mist-100` | `#F0EFE7` | 页面背景、地图外露底色 |

### 3.2 功能色

| 令牌 | 色值 | 用途 |
| --- | --- | --- |
| `--cinnabar-500` | `#A84E3F` | 结束回合、确认出征、危险行动 |
| `--jade-500` | `#365F56` | 当前页、焦点、稳定与成功 |
| `--bronze-500` | `#765527` | 钱粮、官职、少量强调 |
| `--river-500` | `#668794` | 情报、路线、冷色状态 |
| `--warning-500` | `#A3483A` | 风险、损失与警告 |

整体明度以亮宣纸为主，深墨色只承担文字、细线和短暂转场。朱砂只用于少量决定性操作，不作为大面积背景；青玉色承担当前状态和交互焦点。

势力颜色单独配置，并检查相邻势力对比。所有状态同时配合图标或文字。

## 4. 字体与排版

- 全局中文使用粗楷体，标题、按钮和正文保持统一的汉末书卷气质；默认字重为 700。
- 系统优先选择 `Kaiti SC` / `STKaiti` / `KaiTi`，缺字和数字由项目内置 `Noto Sans SC` 回退，避免跨平台乱码。
- 标题通过字号、留白和墨色建立层级，不再混用现代黑体；密集数据仍须保证清晰行距。
- 数字使用稳定宽度的回退字形，避免资源变化时宽度跳动。

字号建议：

| 层级 | 字号 | 行高 |
| --- | ---: | ---: |
| 场景标题 | 32 px | 40 px |
| 面板标题 | 22 px | 30 px |
| 卡片标题 | 17 px | 24 px |
| 正文 | 15 px | 23 px |
| 辅助信息 | 13 px | 19 px |

正文不低于 14 px。长战报行宽控制在 38 至 46 个中文字符。

## 5. 间距与尺寸

使用 4 px 基准：`4 / 8 / 12 / 16 / 24 / 32 / 48`。

- 顶部资源栏：64 px。
- 左右侧主要面板：320 至 400 px。
- 主要按钮高度：40 至 44 px。
- 次级按钮高度：34 至 38 px。
- 点击目标最小尺寸：40×40 px。
- 卡片圆角：6 px，避免现代应用式的大圆角。

## 6. 材质与面板

面板使用暖白宣纸底、轻微纸纹、淡古铜细边和柔和投影。地图上的浮层保持接近实色的浅宣纸，复杂背景下可加局部白色水墨晕染，但避免整屏毛玻璃。

大型面板可在左上使用松柏、右下使用桃花枝作为角花。角花必须由生图工具产出，并严格保持纯黑、灰、留白的水墨风，不使用彩色花叶；显示时控制在 5% 至 13% 不透明度，置于内容下方且不承载交互或正式信息。

推荐层级：

- 一级面板：暖白宣纸，不透明度 96% 至 99%，承载表单和长信息。
- 二级浮层：浅宣纸，不透明度 92% 至 97%，承载地图摘要。
- 当前标签：低饱和青玉实色底配暖白文字。
- 标签和徽记：接近实色，确保小尺寸可读。

## 7. 天下地图界面

### 7.1 布局

```text
┌────────────────────────────────────────────────────────┐
│ 势力 / 年月       金钱  粮食  威望        设置 / 回合结束 │
├──────────┬─────────────────────────────┬───────────────┤
│ 任务与    │                             │ 选中城池摘要   │
│ 情报筛选  │        天下地图             │ 太守 / 驻军    │
│          │ 城池、路线、军团覆盖层       │ 钱粮 / 民心    │
│          │                             │ 可执行操作     │
├──────────┴─────────────────────────────┴───────────────┤
│ 最近事件 / 行军消息 / 风险提醒                           │
└────────────────────────────────────────────────────────┘
```

### 7.2 城池标记

- 正常状态显示城名、旗色和规模。
- 悬停显示驻军区间、太守和最近情报时间。
- 被围显示脉冲外环，但不得持续强闪烁。
- 缺粮、叛乱和灾害使用独立图标。
- 城池标记大小在缩放时限制上下限，避免消失或遮满地图。
- 天下底图采用暖白纸色、淡灰水墨山河和低对比地纹，不把高饱和地形色与城池旗色放在同一视觉层级；道路使用深灰蓝/赭色程序线叠加。
- 全国总览时城池标记保持不少于局部视角约 68% 的屏幕尺寸，城名继续使用实色文字与描边，确保城市不会融入地图。

### 7.3 地图状态

- 默认：地形与归属。
- 军事：兵力、路线和补给范围。
- 内政：农业、商业、民心热力。
- 外交：关系和盟友边界。
- 人才：已知在野人才与招募线索。

## 8. 城池与议政界面

`council-hall-bg-v1.png` 用于营造势力中枢氛围，面板覆盖左右两侧，中央桌案可放置当前政策摘要或本月重点。

信息分为四个标签：

- 概览：人口、钱粮、民心、治安、城防和风险。
- 发展：设施、行动槽、建设计划和收益预估。
- 人才：太守、在城武将、任务和关系提醒。
- 军事：驻军、训练、征兵和可组建军团。

所有行动按钮在执行前显示消耗、负责武将、预计结果和持续时间。

## 9. 武将界面

武将卡片强调头像、姓名、官职、五维雷达或短条、忠诚和状态。不要一次展示所有隐藏人格数据。

详情页分为：

- 能力与特长。
- 当前职责和位置。
- 功勋、忠诚和期望。
- 已知关系。
- 近期事件与评价。

忠诚不只用红绿颜色表达，同时显示文字等级和变化趋势。

## 10. 出征准备界面

出征采用分栏工作台：

- 左侧：可选武将和兵力。
- 中央：军团编制、路线、粮草和战术。
- 右侧：战力预估、胜率区间、优势与风险。

胜率不得显示虚假的单点精确值。低情报时显示如“35%～60%，可信度低”，并说明未知守军或战术可能导致变化。

确认按钮旁必须显示预计到达时间和粮草剩余。

## 11. 战斗结果界面

第一版没有战场镜头。点击结算后进入战报场景，以 `expedition-result-bg-v1.png` 为底图。

```text
┌────────────────────────────────────────────────────────┐
│                     战役结果：胜利                      │
│                 陈留军 12,480  vs  汝南军 9,760         │
├───────────────────────┬────────────────────────────────┤
│ 己方损失 / 敌方损失    │ 影响胜负的关键因素             │
│ 伤兵 / 溃逃 / 俘虏     │ + 主将统率                     │
│ 缴获与城防损失         │ + 粮草充足                     │
│ 武将状态               │ - 雨后道路泥泞                 │
├───────────────────────┴────────────────────────────────┤
│ 三段式战报：接战 → 转折 → 结果                          │
│                                      [查看详情] [继续]  │
└────────────────────────────────────────────────────────┘
```

评分条使用相同标尺，默认先展示结果，再依次显现损失和关键因素。总动画控制在 2.5 秒内，并允许立即跳过。

## 12. 动效

- 面板出现：160 至 220 ms，轻微位移和淡入。
- 地图选中：120 ms 外环扩散。
- 数值变化：300 ms，配合正负号和颜色。
- 回合推进：场景保持，使用月份卷轴或天光变化，控制在 1 秒左右。
- 战报：分阶段揭示，但不强迫等待。

启用“降低动态效果”时，移除缩放、视差、持续脉冲和大幅位移。

## 13. 声音方向

- 地图：风、鸟鸣、远处市集和极轻的古乐氛围。
- 城市：根据繁荣和治安变化人声密度。
- 操作：木、纸、玉、铜等克制的材质反馈。
- 战报：鼓点、号角和环境声，不播放长战斗演出。

当前全局背景音乐由两首原创纯器乐组成：《山河未定》以古琴、洞箫、琵琶、低鼓、编钟和少量二胡构成约 72 BPM 的克制中国风氛围；《烽火长歌》v3 基于 v2 的旋律和结构改编，仅以琵琶与古筝呈现苍茫、回望、英雄散尽的乱世史诗感，并明确排除演唱、吟唱、合唱、哼唱、念白和人声采样。两首曲目默认以低音量顺序循环，并受顶栏声音总开关控制。

## 14. 响应式策略

- 设计基准为 1920×1080。
- 1366×768 时侧栏改为可收起抽屉，保留地图主体。
- 2560×1440 时限制面板最大宽度，不让文字行过长。
- 第一版不为手机竖屏设计完整玩法；小于 1024 px 显示不支持提示或只读概览。

## 15. 生图资产规则

- 背景图不得生成中文文字、按钮、数值、商标或水印。
- 构图必须预留界面安全区和清晰焦点。
- 同一场景的迭代使用版本后缀，不覆盖源图。
- 生成资产进入项目后记录用途、提示词、日期和状态。
- 人物立绘应保持同一光向、裁切和材质风格；正式批量生成前先做 3 人风格测试。
- 不直接模仿在世艺术家的个人风格，不使用未经授权的商业游戏素材作为训练参考或拼贴素材。

## 16. 设计验收

- 去掉背景图后，全部功能和信息仍然完整。
- 复杂背景上文字对比度仍可读。
- 玩家能在 5 秒内定位当前年月、资源、选中城市和回合结束按钮。
- 战前能理解胜率为什么高或低，战后能理解胜负为何发生。
- 所有危险操作可撤销草稿或二次确认。
- 颜色视觉障碍下仍能区分势力和状态。
